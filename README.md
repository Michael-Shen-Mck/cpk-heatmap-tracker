# CPK Heatmap 平台

这是一个基于 Streamlit 和 SQLite 的轻量平台，用于录入不同规格、轧线、班组、批次的捆重数据，并按负差规则计算 Cpl 过程能力热力图。

## 启动方式

```powershell
python -m pip install -r requirements.txt
python -m streamlit run app.py
```

启动后在浏览器中打开 Streamlit 提供的本地地址。

## 最简单云端部署

适合先做成你和客户都能打开的网址。

1. 把整个项目上传到一个 GitHub 仓库。
2. 打开 Streamlit Community Cloud：https://share.streamlit.io
3. 选择 GitHub 仓库，入口文件填 `app.py`。
4. 部署完成后，把生成的网址发给客户即可。当前版本不设密码，客户点开网页就能直接使用。

注意：这个简单版仍使用 SQLite 文件数据库，适合演示、小范围试用和阶段性数据录入。正式长期使用时，建议定期在“数据明细与导出”中导出 CSV 备份；如果要保证云端数据长期稳定保存，后续应升级为 PostgreSQL 云数据库。

云端空库首次启动时，系统会自动生成 8 个规格和 2026 年 6 月演示数据，方便客户打开链接后直接看到热力图效果。客户学会后，可以在“数据明细与导出”里的“数据管理”区域删除演示数据或清空全部捆重数据。

## 使用流程

1. 进入“规格配置”，新增规格、目标捆重和允许负差。
2. 进入“数据录入”，选择规格、日期、轧线、班组和批号，粘贴每捆重量并提交，或下载 Excel 模板后批量导入。
3. 进入“过程能力热力图”，选择月份、轧线和班组，查看每个规格每天的 Cpl 表现。
4. 进入“数据明细与导出”，筛选原始数据、批次汇总和指标，并导出 CSV。

## 计算口径

- 偏差：`实际捆重 - 目标捆重`
- 允许负差下限：`-允许负差`
- Cpl：`(平均偏差 - 允许负差下限) / (3 * 偏差标准差)`
- 默认至少 5 条样本才计算 Cpl。

热力图颜色规则：

- `Cpl >= 1.33`：良好
- `1.00 <= Cpl < 1.33`：关注
- `Cpl < 1.00`：不足
- 无数据或样本不足：空白
